# 演示脚本 / Demo Script

Use this flow for a GitHub README video, classroom presentation, or project defense.  
可用于 GitHub README 视频、课堂展示或项目答辩。

## 60 秒演示 / 60-second Demo

1. 打开 `index.html`。 / Open `index.html`.
2. 展示首屏并说明目标：把隧道监测、支护时序、周边建筑和施工参数放在一个可解释仪表盘里。 / Show the hero section and explain the goal: tunnel monitoring, support timing, surrounding buildings, and construction parameters in one explainable dashboard.
3. 切换到 `山岭隧道断层破碎带 / Mountain tunnel fault fracture zone`。 / Switch to the mountain tunnel fault scenario.
4. 指出风险分数、响应等级和主控因素。 / Point out the risk score, response level, and key drivers.
5. 增加 `地下水位变化 / Groundwater Change` 或 `支护滞后 / Support Lag`。 / Increase groundwater change or support lag.
6. 展示风险贡献条和建议如何更新。 / Show how risk contribution bars and recommendations update.
7. 导入 `data/monitoring-sample.csv`。 / Import `data/monitoring-sample.csv`.
8. 导出 JSON 和 Markdown 报告。 / Export JSON and Markdown reports.
9. 介绍路线图：BIM/IFC、监测曲线、岩土资料、报告生成。 / Close with the roadmap: BIM/IFC, monitoring curves, geotechnical data, and report generation.

## 讲解重点 / Key Talking Points

- 项目是原创且零依赖。 / The project is original and zero-dependency.
- 它参考开源土木项目的方向，但没有复制代码。 / It uses open-source civil engineering projects as inspiration, not copied code.
- 它强调可解释性：工程师可以追溯每个风险分数的来源。 / It is explainable: engineers can trace every score back to input factors.
- 静态架构使其容易发布到 GitHub Pages。 / The static architecture makes it easy to publish on GitHub Pages.
